# Perubahan 2026-06-07 — Copy Engine Auto-Deploy + Re-scan Existing Wallets

## Latar Belakang

Copy engine selama ini hanya **display/pajangan**. Setelah `analyzePositionForCopy()` menghasilkan `action: "COPY"` dengan confidence ≥ threshold, signal disimpan ke `copy-signals.json` dan ditampilkan di dashboard kolom **Wallet**, tapi **tidak pernah benar-benar di-deploy**.

Selain itu, wallet yang sudah pernah menghasilkan COPY signal tidak pernah di-scan ulang — setiap cycle hanya scan wallet dari ranking terbaru.

## Akar Masalah

Di `copy-engine/position-monitor.js`, fungsi `runCopyEngineCycle()` hanya:
1. Scan wallet ranking → dapat posisi
2. Evaluasi via decision engine
3. Simpan signal ke JSON
4. Kirim Telegram
5. **Selesai — tidak lanjut ke deploy**

`buildDeployArgs()` sudah membuat parameter deploy lengkap (pool, amount, bins, dll), tapi tidak ada kode yang memanggil `deploy_position`.

## Perbaikan — 2 Fitur

### Fitur 1: Auto-Deploy COPY Signals

Tambah loop deploy di `runCopyEngineCycle()` setelah signals dikumpulkan:

```
for setiap signal dengan action === "COPY":
  panggil executeTool("deploy_position", signal.deployArgs)
  handle: success, blocked, error, exception
  hitung deployedSignals.length
```

### Fitur 2: Re-scan Wallet dari Existing COPY Signals

Setiap cycle, copy engine sekarang juga mengambil wallet dari `copy-signals.json` yang sebelumnya pernah menghasilkan `action: "COPY"`, lalu menggabungkannya dengan wallet ranking terbaru.

**Alur:**

```
getTopWalletEntries() → topWallets (dari ranking snapshot)
getRecentCopySignals({ action: "COPY" }) → prevCopySignals (dari copy-signals.json)
  ↓
Merge + deduplicate berdasarkan address
  ↓
allWallets = [...topWallets, ...prevWalletEntries]
  ↓
Scan semua wallet:
  - Ranked wallets: dedupe normal (hasRecentCopySignal)
  - Prev COPY wallets: skipDedupe=true (tidak kena duplicate check)
  ↓
Deploy signal yang lolos decision engine
```

**Kenapa skipDedupe untuk prev COPY wallets?**
Tanpa `skipDedupe`, wallet dari existing COPY signals akan ke-detek duplikat oleh `hasRecentCopySignal()` karena signal lama masih dalam TTL (12 jam) → posisi mereka di-skip → tidak pernah di-deploy.

## Detail Implementasi

### Merge wallet dari existing COPY signals

```javascript
const prevCopySignals = getRecentCopySignals({ action: "COPY", limit: 50 });
const existingWalletSet = new Set(topWallets.map((w) => w.address));
const prevWalletEntries = [];
for (const signal of prevCopySignals) {
  if (!signal.wallet || existingWalletSet.has(signal.wallet)) continue;
  existingWalletSet.add(signal.wallet);
  prevWalletEntries.push({
    address: signal.wallet,
    score: Number(signal.walletScore) || 50,
    grade: signal.walletGrade || "B",
    rank: Number(signal.walletRank) || 999,
    label: signal.walletLabel || signal.wallet.slice(0, 8),
    type: "copy_signal_wallet",
    source: "prev_copy_signal",
    confidence: Number(signal.walletScore) || 50,
  });
}
const allWallets = [...topWallets, ...prevWalletEntries];
```

### Deploy loop

```javascript
const deployedSignals = [];
for (const signal of signals) {
  if (signal.deployArgs && signal.action === "COPY") {
    try {
      const deployResult = await executeTool("deploy_position", signal.deployArgs);
      if (deployResult?.would_deploy || deployResult?.success) {
        deployedSignals.push({ ...signal, deployResult });
      } else if (deployResult?.blocked) {
        log("copy_engine", `Deploy blocked: ... — ${deployResult.reason}`);
      } else if (deployResult?.error) {
        log("copy_engine", `Deploy error: ... — ${deployResult.error}`);
      }
    } catch (err) {
      log("copy_engine_error", `Deploy exception: ... — ${err.message}`);
    }
  }
}
```

## Alur Baru (Lengkap)

```
getTopWalletEntries() ──┐
                        ├── merge + deduplicate → allWallets
getRecentCopySignals() ─┘
  ↓
Scan posisi tiap wallet:
  - Ranked wallets: dedupe normal (hasRecentCopySignal)
  - Prev COPY wallets: skipDedupe=true
  ↓
Decision Engine → action: "COPY" / "HOLD" / "SKIP"
  ↓
action === "COPY" & confidence >= threshold
  ↓
executeTool("deploy_position", deployArgs)  ← BARU! Auto-deploy
  ↓
Success / Blocked / Error → log + summary.deployed
```

## Risiko & Safety

- **Dry Run**: Bot masih `dryRun: true` — deploy hanya simulasi
- **Sequential**: Loop `for...of` (tidak parallel) — aman dari race condition
- **Error per signal**: Setiap deploy punya `try/catch` sendiri
- **Already in position**: `deploy_position` sudah handle duplicate detection
- **Dedupe terpisah**: Wallet ranking tetap kena dedupe, hanya prev COPY wallets yang di-skip dedupenya
- **Interval 15 menit**: Copy engine cron jalan tiap 15 menit

## Update — Rank & Floor Fix untuk Prev COPY Wallets (v3.1)

### Masalah
Wallet dari existing COPY signals dapat `rank: 999` (hardcode) → rankFactor mentok di 0.5 → amount yang direkomendasikan sangat kecil (0.04-0.09 SOL) → kena block oleh safety gate `minDeploy = 0.13 SOL`.

### Perbaikan

**1. Default rank 10 (bukan 999)**
```javascript
// Sebelum:
rank: Number(signal.walletRank) || 999,
// Sesudah:
rank: Number(signal.walletRank) || 10,
```

**2. Floor amount untuk copy_signal_wallet**
`recommendAmount()` sekarang terima parameter `options.useFloor`. Jika true dan calculated amount < `base` (deployAmountSol), return `base`.

```javascript
function recommendAmount(walletEntry, options = {}) {
  const amount = Math.round(Math.min(max, base * rankFactor * scoreFactor) * 1000) / 1000;
  // Floor: untuk wallet existing COPY signals, pastikan amount minimal = deployAmountSol
  if (options.useFloor && amount < base) {
    return Math.round(base * 1000) / 1000;
  }
  return amount;
}

// Panggilan:
recommendAmount(walletEntry, { useFloor: walletEntry.type === "copy_signal_wallet" })
```

### Dampak

| Wallet | Sebelum | Sesudah | Lolos? |
|--------|---------|---------|--------|
| Prev COPY (rank=10, score=50) | 0.046 SOL ❌ | **0.13 SOL** ✅ floor |
| Prev COPY (rank=5, score=70) | 0.092 SOL ❌ | **0.13 SOL** ✅ floor |
| Ranked wallet (rank=3, score=80) | 0.14 SOL ✅ | 0.14 SOL ✅ (tidak kena floor) |
| Ranked wallet (rank=10, score=50) | 0.065 SOL ❌ | 0.065 SOL ❌ (ranked — kena safety gate) |

Catatan: Ranked wallets tidak kena `useFloor` — hanya wallet dari existing COPY signals yang dapat perlindungan ini.
