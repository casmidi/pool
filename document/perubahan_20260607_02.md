# Perubahan 2026-06-07 (02) — Bug Fix Dashboard

## Ringkasan
Memperbaiki **2 bugs kritis** yang menyebabkan **Opportunity Watchlist kosong** di dashboard utama.

## Bug #1: `config is not defined` di Route `/api/pools` & `/api/copy-signals`

### Root Cause
Variabel `config` digunakan di fungsi `enrich()` dan `enrichSignal()` untuk membaca `config.executionIntelligenceMode`, tapi **tidak pernah didefinisikan** di scope route handler tersebut.

### Route Terdampak
| Route | Problem | Fix |
|-------|---------|-----|
| `GET /api/pools` | `const config` tidak ada | Ditambah `const config = readJSON(PATHS.userConfig, {});` |
| `GET /api/copy-signals` | `const config` tidak ada | Ditambah `const config = readJSON(PATHS.userConfig, {});` |

### Dampak
- `/api/pools` selalu **crash** dengan `ReferenceError: config is not defined`
- Semua data yang dikirim ke Opportunity Watchlist jadi kosong
- Dashboard hanya menampilkan "No eligible pools right now"

---

## Bug #2: `getAllLogLines()` Baca Full File Log (26K+ Baris)

### Root Cause
Fungsi `getAllLogLines()` membaca **seluruh isi 2 file log terakhir** tanpa batas.

### Sebelum (Problem)
```js
function getAllLogLines() {
  // Membaca FULL file — 2 file × 26.119 baris = 52.000+ baris
  const content = fs.readFileSync(filePath, "utf8");
  lines.push(...content.split("\n").filter(Boolean));
}
```

### Sesudah (Fix)
```js
function getAllLogLines() {
  const MAX_LINES_PER_FILE = 500;
  // Untuk file > 100KB, baca 64KB dari akhir file
  // Untuk file kecil, baca normal lalu slice(-500)
  const chunkLines = content.split("\n").filter(Boolean);
  lines.push(...chunkLines.slice(-MAX_LINES_PER_FILE));
}
```

### Dampak
- Waktu loading `extractPoolsFromLog()` turun drastis
- Response API pools jadi cepat (tidak timeout)
- Semua data tetap akurat karena pool terbaru selalu ada di baris paling akhir
