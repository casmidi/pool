# Perubahan 2026-06-09 — Smart Wallet Observer Fix + Forensic Integration

## Latar Belakang
Observer tidak pernah mengakumulasi `samples` karena 2 bug:

1. **`sourceWallet` tidak diteruskan** dari executor.js ke pnl_tracker saat deploy — semua closed trade punya `source_wallet: null` → observer tidak bisa link trade ke wallet address
2. **Observer interval crash diam-diam** — `run()` tidak punya try/catch, error bikin setInterval mati tanpa log

## Perubahan

### 1. executor.js — pass `sourceWallet` ke recordDeploy
**File:** `tools/executor.js:901-903`

Tambah field ke recordDeploy:
```js
sourceWallet: args.source_wallet ?? null,
sourceWalletScore: args.source_wallet_score ?? null,
sourceWalletGrade: args.source_wallet_grade ?? null,
```

**Impact:** Semua deploy dari copy engine sekarang punya `source_wallet` di trade record → observer bisa link closed trade ke wallet address.

### 2. smart-wallet-observer.js — error handling
**File:** `smart-wallet-observer.js:276-285`

- Ganti `var run` blak-blakan dengan `_runOnce()` yang di-wrap try/catch
- Error ditulis ke stderr (`console.error`) agar tertangkap PM2 log
- Interval tetap jalan meskipun satu siklus gagal

### 3. pnl_tracker.js — export `loadStore()`
**File:** `lib/pnl_tracker.js:764-766`

Export fungsi `loadStore()` yang wraps internal `load()` — untuk akses langsung dari observer/dashboard ke seluruh trade store.

## Status Saat Ini
- Observer sudah berjalan dan update file (`generatedAt: 2026-06-09T07:53:52Z`)
- `sampleCount` masih 0 karena 72 closed trade existing tidak punya `source_wallet`
- Samples akan terisi otomatis setelah ada copy-engine deploy baru yang close

## Data Flow Setelah Fix
```
wallet (ranking_db)
  → position-monitor.js: deployArgs.source_wallet = wallet.address
    → executor.js: args.source_wallet
      → pnl_tracker: trade.source_wallet = wallet.address
        → observer: match trade.source_wallet → increment wallet.samples
```
